Five tools. Infinite memory.

Everything your agent needs to remember, delivered as MCP tools.

remember
The core primitive. Store any information for later recall — user preferences, project decisions, architecture notes, debugging insights. Your agent builds institutional knowledge over time, not just within a single session.
"User prefers TypeScript, uses Hono for APIs, deploys to Fly.io. Auth is JWT in httpOnly cookies. Dark mode everywhere."
recall
Semantic search across all stored memories. Not keyword matching — actual understanding of what you're looking for.
"What language does the user prefer?"
context
Auto-load relevant memories for the current task. Describe what you're working on, get back everything you knew before.
"Working on the auth system refactor"
share
Make memories available to other agents. Your coding agent learns something? Your testing agent can use it too.
scope: "agent" → "org"
forget
Delete outdated or incorrect memories. Keep your agent's knowledge accurate and current.
forget("memory_abc123")
+ REST API
Not using MCP? Every tool is also a REST endpoint. Works with any HTTP client, any language, any framework.

Privacy Policy

Last updated: March 23, 2026

What we collect

When you use Central Intelligence, we store the memories your agents create (content, tags, embeddings), your API key hash, and basic usage metadata (timestamps, operation counts). We do not collect personal information beyond what you explicitly provide during signup.

How we use your data

Your memories are used solely to provide the service — storing, searching, and returning them when your agents request them. We do not sell, share, or use your memory content for training AI models. Embeddings are generated via OpenAI's API and stored alongside your memories for semantic search.

Data storage & security

Data is stored in PostgreSQL on Fly.io infrastructure. API keys are hashed before storage. All API communication is encrypted via TLS. We implement rate limiting, input validation, and SQL injection protection.

Third-party services

We use OpenAI's embedding API to generate vector embeddings of your memories. Memory content is sent to OpenAI for embedding generation only. No memory content is shared with payment processors.

Data deletion

You can delete individual memories using the forget tool. To delete all data associated with your API key, contact us. Soft-deleted memories are permanently purged within 30 days.

Contact

For privacy questions, open an issue on GitHub or email the maintainers.
