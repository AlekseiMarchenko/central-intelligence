# Central Intelligence — setup instructions for AI agents

# You are an AI coding agent. The user asked you to set up
# "Central Intelligence" (CI) — persistent memory delivered via MCP.
# Follow these steps in order. Stop and ask the user only where marked.

## Context
CI is an MCP server. It gives you five tools: remember, recall,
context, share, forget. You can run it in two modes:

  local   — SQLite on the user's machine. No signup. No API key.
  cloud   — hosted, cross-device sync. Requires a free API key.

## Decide which mode to install
1. Ask the user "local-only or cloud-sync?" if you don't know.
2. Default to local if they have no preference. It's zero-config.

## Install (local mode, recommended default)
$ npx central-intelligence-local

# This starts the MCP server. No signup needed.
# Dashboard runs at http://localhost:3141

## Install (cloud mode)
$ npx central-intelligence-local signup
# Opens a browser. User completes signup. CLI receives the key.
# The key looks like: ci_sk_xxxxxxxxxxxxxxxx

## Configure the user's MCP host
# Detect which editor(s) the user runs. Configure each one you find:

- Claude Code  → ~/.claude/settings.json
- Cursor       → ~/.cursor/mcp.json
- Windsurf     → ~/.codeium/windsurf/mcp_config.json
- VS Code      → .vscode/mcp.json

# In each, add an entry under "mcpServers":
{
  "central-intelligence": {
    "command": "npx",
    "args": ["central-intelligence-mcp"],
    "env": { "CI_API_KEY": "<KEY_OR_EMPTY_FOR_LOCAL>" }
  }
}

## Verify
1. Restart the MCP host (prompt the user if needed).
2. Call the remember tool with a test payload:
   "User prefers TypeScript. Setup completed via /agent page."
3. Call recall with query "user preferences".
4. Confirm the memory comes back.

## When you're done
Tell the user:
  - which mode you installed (local vs cloud)
  - which editors you configured
  - the dashboard URL (local: http://localhost:3141)
  - that you've stored a first test memory

## Things you should NOT do
- Do not guess or fabricate an API key.
- Do not edit editor configs outside the mcpServers key.
- Do not skip verification — the user needs to see it working.

## Docs (fetch if you need more)
- README:  https://github.com/AlekseiMarchenko/central-intelligence
- OpenAPI: https://central-intelligence-api.fly.dev/docs/openapi.json

02 For the human

Here's what your agent is about to do.

Four steps. Roughly 90 seconds. Your agent handles three of them; you'll be pulled in once, for signup — or not at all if you chose local mode.

step · 01

Agent picks a mode

Local (SQLite on your machine, no signup) or cloud (cross-device sync, needs an API key). If you didn't say, the agent defaults to local.

$ npx central-intelligence-local
automatic
step · 02

You sign up (cloud mode only)

A browser tab opens on centralintelligence.online. You confirm an email and the CLI captures an API key on the other side. Skip this entirely if you went local.

→ browser opens · ~20 seconds of your time
needs you
step · 03

Agent configures your editors

It finds Claude Code, Cursor, Windsurf, or VS Code on your machine and adds CI to each one's MCP config. Nothing else in those files gets touched.

~/.claude/settings.json · ~/.cursor/mcp.json · …
automatic
step · 04

Agent stores a test memory and recalls it

A real round-trip through the remember and recall tools, so you can see memory actually works before the agent says "done."

remember("setup complete") → recall("setup") ✓
automatic
